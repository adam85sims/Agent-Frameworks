# Phase 1: Governance Decoupling

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Make governance/ a drop-in audit framework for ANY Python project by replacing hardcoded paths, LM Studio assumptions, and narrow claim patterns with config-driven, pluggable alternatives.

**Architecture:** governance/ imports from common/ for config loading, logging, and data models. Evidence collection reads from config instead of hardcoded paths. Auditor supports multiple LLM backends via a backend abstraction. Claims extraction is extensible via config-defined patterns. The module exposes a clean Python API alongside the CLI.

**Tech Stack:** Python 3.11+, common/ (Phase 0), PyYAML (optional), urllib (stdlib, for LLM API calls).

## Task 2: Decouple auditor.py from LM Studio

**Objective:** Replace hardcoded LM Studio URLs/paths with config-driven backend selection.

**Files:**
- Modify: `governance/auditor.py`
- Modify: `governance/auditor.yaml`

**Changes:**
1. Replace `LM_STUDIO_URL`, `LMS`, `LMSTUDIO_MODELS_DIR` constants with config values from auditor.yaml
2. Add a `backend` section to auditor.yaml:
   ```yaml
   backend:
     type: "openai-compatible"
     url: "http://localhost:1234/v1/chat/completions"
     lms_binary: "~/.lmstudio/bin/lms"
   ```
3. `is_model_on_disk()` and `is_model_loaded()` should use the configured backend
4. `load_model()` should be optional — skip if backend doesn't support explicit loading
5. Add env var overrides: `AGENT_FW_AUDITOR_URL`, `AGENT_FW_AUDITOR_MODEL`

**Verification:** auditor.py loads without error and reads config from auditor.yaml.
